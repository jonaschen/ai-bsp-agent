# Log Pending List — Real-World Validation (Item #20)

This document tracks every log file that must be generated or crafted to
validate the 11 BSP diagnostic skills against realistic inputs.

## How to use this list

1. Generate or craft the log (see **Source** column).
2. Run it through the BSP agent: `python cli.py --dmesg <file> [--meminfo <file>]`
3. Compare the agent's output against the **Expected outcome**.
4. Record any discrepancy under **Findings** and update the skill/threshold accordingly.
5. Change **Status** to `PASS`, `FAIL`, or `PARTIAL`.

## Generation methods

| Method | Description |
|---|---|
| `android-avd` | Generated by `emulator_scripts/run-android-emulator.sh` |
| `linux-qemu` | Generated by `emulator_scripts/run-linux-qemu.sh` |
| `aarch64-qemu` | Generated by QEMU `qemu-system-aarch64` (machine `virt`): either Alpine Linux 3.19 aarch64 kernel boot with hardware-specific message injection, or real LK AArch64 (github.com/littlekernel/lk) bootloader output |
| `synthetic` | Manually crafted to cover ARM/hardware-specific patterns the emulator cannot produce |
| `real-hw` | Must come from actual hardware; emulator cannot replicate |
